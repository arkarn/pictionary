# Pictionary MCP App - Agent Documentation

This document explains the architecture, quirks, and current state of the Pictionary MCP App to help future AI agents understand the codebase without needing to read the entire conversation history.

## High-Level Architecture
This project is an **MCP (Model Context Protocol) App**. It consists of two main parts:
1. **The Backend Server (`server.ts`)**: An MCP server that exposes tools and resources.
2. **The Frontend UI (`src/mcp-app.tsx`)**: A React application that is bundled into a single HTML file (`mcp-app.html`) and served by the backend as an MCP resource (`ui://pictionary/mcp-app.html`).

The app uses **Excalidraw** to render drawings generated by an LLM (Gemini) and allows users to guess the word.

## The Core Challenge: Streaming vs. `basic-host`
The original goal was to have the *host LLM* (e.g., Claude Desktop) stream the drawing elements token-by-token directly into the UI using the MCP `ontoolinputpartial` event. 

**However, the primary testing environment (`@modelcontextprotocol/ext-apps/examples/basic-host`) does NOT have an LLM attached to its "Call Tool" button.**
* When a user clicks "Call Tool" in `basic-host`, it sends the JSON payload from the text area instantly as a single `tools/call` request.
* Because there is no LLM generating the JSON live, no partial input events fire.

### The Solution: Simulated Streaming (Animated Rendering)
To work around the lack of host-side streaming in `basic-host` while still providing a "live drawing" experience, we implemented a **Simulated Streaming Architecture**:

1. **Backend Generation**: The `draw_pictionary` tool in `server.ts` calls the Gemini API directly to generate the complete Excalidraw JSON array in one shot.
2. **Instant Delivery**: The server returns the final, complete array to the frontend in its `CallToolResult`.
3. **Animated Frontend Rendering**: The React frontend (`mcp-app.tsx`) intercepts the full array. Instead of rendering it immediately, it uses an interval to push the elements into the `Excalidraw` component one by one (e.g., every 300ms) using `excalidrawApiRef.current.updateScene()`.

To the user, this looks exactly like real-time LLM streaming, but it bypasses the host entirely.

## Key Quirks & Implementation Details

1. **`registerAppResource` is Critical**: In `server.ts`, we *must* register the UI resource `ui://pictionary/mcp-app.html`. If this is missing, `basic-host` literally doesn't know what HTML to load and the UI area will be completely blank.

2. **Strict Content Security Policy (CSP)**: The `basic-host` enforces extremely strict CSP rules on the iframe running the app.
   * `Excalidraw` tries to dynamically load fonts from `https://esm.sh`.
   * **Fix**: We must inject CSP metadata into the resource registration in `server.ts`:
     ```typescript
     _meta: { ui: { csp: { resourceDomains: ["https://esm.sh"] } } }
     ```

3. **Excalidraw Rendering Quirks**:
   * `Excalidraw` only reads the `initialData` property *once* when it mounts.
   * To update the drawing dynamically (for the simulated streaming), you MUST capture the `excalidrawAPI` instance via the `excalidrawAPI={(api) => ...}` prop and call `api.updateScene({ elements: ... })`.
   * The `canvasKey` state variable is used to force a full re-mount when wiping the board clean for a new game.

4. **Gemini API Model Quirks**:
   * We wanted to use `gemini-3.0-flash` or `gemini-3.1-pro-preview` for better spatial reasoning.
   * Ensure the model identifier you use actually exists on the public `v1beta` endpoint, otherwise the API silently fails and returns an empty drawing array `[]`.

## Making Changes

* **Changing the UI**: If you edit `src/mcp-app.tsx` or `src/global.css`, you MUST run `npm run build` to regenerate the `dist/mcp-app.html` bundle.
* **Changing the Server**: If you edit `server.ts` (or the backend logic), you must restart the server process. Wait for the port to clear (`lsof -ti :3001 | xargs kill -9 2>/dev/null`) and then run `npm run serve`.
* **Testing in basic-host**: Start the `basic-host` (on port 8080), select `pictionary-server`, and call the `draw_pictionary` tool leaving the Input JSON empty.

## Tech Stack
* Backend: TypeScript, `@modelcontextprotocol/sdk`, `@google/generative-ai`
* Frontend: React, Vite (`vite-plugin-singlefile`), `@excalidraw/excalidraw`
